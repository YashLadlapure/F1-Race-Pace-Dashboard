# F1 Race Pace & Pit Strategy — Project Story

This project started with a single CSV file called f1.csv. The file had 372,493 rows of Formula 1 lap data going from 2010 all the way to 2026. Each row was one lap driven by one driver in one race. Columns included the race year, driver ID, constructor ID, average lap time in seconds, best lap time, total laps, pit laps, and pit percentage.

The first thing I did was load it in Python using Pandas and just look at the data. `df.info()`, `df.describe()`, `df.isnull().sum()` — the basics. There were some null values in the pit-related columns which made sense since not every lap has a pit stop. The bigger issue was outliers in the lap time columns. Some values were way too low or way too high to be real F1 lap times. An F1 lap is generally between 60 and 120 seconds depending on the circuit, so I filtered anything outside 60–150 seconds and that cleaned things up nicely.

After cleaning, I did feature engineering. I created two aggregated summary tables using groupby — one grouped by race year and race ID to get race-level stats, and another grouped by driver ID and year to get driver-level stats. The aggregations used mean for average lap time, min for best lap time, and sum for laps and pit laps. These two tables became the actual data sources loaded into Tableau.

For EDA I generated seven plots in Python — a line chart of average lap time per season, a histogram of lap time distribution, a boxplot by year, a correlation heatmap, a bar chart of top 15 drivers, a pit stop trend line, and a scatter of best lap per season. The plots confirmed what I suspected: F1 cars got noticeably faster from 2010 through 2019, then there was a plateau around 2020–2022 when new ground-effect regulations came in and teams needed time to develop the cars.

Then I moved to Tableau Public. I connected both CSVs as data sources and built four sheets. The first was a line chart of average lap time by season — Race Year on columns, AVG(Avg Lap Sec) on rows, colored coral red. The second was a horizontal bar chart of the top 15 drivers by average lap time — filtered to top 15 by measure, sorted ascending. The third was a dual-axis line chart showing both average lap time and average pit percentage on the same chart with synchronized axes — this one required adding a second measure to the rows shelf, right-clicking to set dual axis, then syncing the axes so both lines were readable. The fourth was originally a line chart of best lap per season but I changed the mark type to Circle to make it a dot/scatter chart.

The dashboard was assembled as a tiled 2x2 grid. I added a title text box at the top, two KPI text boxes showing 91.2 average lap time and 372,493 total laps, and a subtitle bar with project details. The background was set to dark navy to give it a clean, professional look.

There were a few things that went wrong. The KPI numbers were getting clipped because the text containers were too small for the font size I was using. I resized the containers and that fixed it. The bigger issue was the Best Lap Per Season chart being cut off on the right side — it only showed data up to around 2020 and the dots for 2021–2026 were running off the edge. The root cause was the dashboard canvas was set to 1000px wide which wasn't enough for two side-by-side charts to render fully. I went into the Dashboard panel, changed the canvas width from 1000px to 1200px, and also applied Entire View on all sheets. After that the full 2010–2026 range showed correctly.

Another issue was the chart titles being nearly invisible. They showed as a very dark gray that blended into the navy background. I went into Format > Title on each sheet and changed the color from #333333 to white (#FFFFFF) with Bold applied. Did this for all four sheets.

The final dashboard has four charts, two KPIs, a title, and a subtitle. It's published live on Tableau Public. The data pipeline goes from raw f1.csv through Python cleaning and aggregation to two clean CSVs, then into Tableau for visualization.
